DESCRIPTION
TCP connection
Server
Server generates a thread that listens for communication when a client is connected. The communication is based on TCP connection.
Client
Clients are connected to the Server to communicate with each other. Each time when a client perform an action in the program, a Doc that includes a type will be sent over to the Server. The Server then sends out the Doc to all threads depending on its type, and there is a listner on the Clients’ side that receives and analyzes commands to generate different actions.

DESIGN
We are using two simple user interfaces. The first one is the user interface for the login process. It’s clear for the user what fields to fill in and what button to press to continue to the project. The second one is the interface for the main functions and code editing. It contains a list with classes and a chat area at the sides and then a main editing area in the middle. The interface contains large buttons that clearly shows their functionality.
